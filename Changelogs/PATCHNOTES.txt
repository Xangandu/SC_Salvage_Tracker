VERSION 0.10.0 Alpha | Build 2026.06 | Codename: Host-Client Vernetzung
------------------------------------------------------------------------

NEU — HOST-CLIENT / NETZWERK
• Host-Client-Architektur: Ein Host hält die SQLite-Datenbank, Clients
  verbinden sich per TCP/JSON (Port 47890, optional TLS)
• Modul network/: HostServer, ClientConnection, RemoteDatabase, RPC-Proxy
• Verbindungsassistent: Standalone / Host / Client (Einstellungen → Vernetzung)
• Authentifizierung: Beitrittscode (Gast) oder Host-Benutzerkonto (volle Rechte)
• Client-Sitzungs-UI: Aktive Host-Sitzung wählen, Materialfelder nach Schiff
  der gewählten Sitzung (kein eigenes „Sitzung starten“ am Client)
• Navigationsleiste: Live-Status ◆ HOST / ◆ CLIENT (IP, Port, Code, Clients)
• Einstellungen → Tab „Vernetzung“: Host starten/stoppen, Assistent
• Live-Sync: Änderungen vom Client aktualisieren Host-UI und umgekehrt
• Host-Anzeige aktualisiert sich bei Start/Stop des Servers

STARTABLauf
• Anmeldung zuerst (auch Erststart admin/admin), danach optional Assistent
• Standard: Solo/Standalone — Vernetzung erst bei Bedarf (Freund joint später)
• Assistent nach Login optional aktivierbar (Checkbox im Assistenten)

DATENBANK / KONFIG
• Schema 020_network_settings.sql — network_mode, network_host_port, TLS, Code
• dependency: cryptography (TLS-Zertifikate für Host)

TESTS
• scripts/network_workflow_test.py — Materialfluss über RPC als Gast-Client
• scripts/integration_test_flow.py — lokaler End-to-End-Test (unverändert)

BEKANNT / OFFEN
• Client-Modus erfordert derzeit Neustart nach Assistenten-Wechsel (Standalone/Host)
• Als Gast gespeicherte Client-Verbindung wird beim Start still wiederhergestellt
  (ohne Assistent); Benutzer-Login erfordert erneute Anmeldung in Einstellungen
• Client: Dashboard-Layout wird lokal gespeichert (nicht per RPC auf dem Host)
• Abmelden: Host-Server stoppen, Client trennen, zurück zum Login (kein Auto-Reconnect)
• Sitzung → „Aktive Sitzung“: Schiff/Status als reine Anzeige (kein Eingabefeld)
• Einstellungen → Vernetzung: „Verbundene Clients“ aktualisiert sich live
• Kein mDNS (Host im LAN automatisch finden)
• Kein Offline-Puffer bei Verbindungsabbruch am Client
