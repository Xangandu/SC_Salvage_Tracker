SC SALVAGE TRACKER – PROJEKT-ZUSAMMENFASSUNG
==============================================

Stand: Version 0.16.7 Beta | Build 2026.18 | Codename: POLISH AND PERFORMANCE


1. PROJEKTZIEL
--------------
Der SC Salvage Tracker ist ein MobiGlas-inspiriertes Desktop-Programm
(Python / PySide6 / SQLite) zur Verwaltung von Star-Citizen-Salvage-
Operationen: Sessions, Materialien, Raffinerie, Verkäufe, Kosten und
Crew-Auszahlungen.


2. ARCHITEKTUR
--------------

2.1 Altes Modell (bis v0.8.0) — ENTFERNT AUS UI
    Session → session_materials → Raffinerie → Verkauf
    • Material direkt an Session gebunden
    • Keine zentrale Nachverfolgung einzelner Materialmengen

2.2 Aktuelles Modell (ab v0.8.5, erweitert v0.16.0)
    Session → Material Batch → Schiff-Lager (IN_SHIP) → Station-Stockpile (STORED)
              ↓
         Refinery Job → Refinery Job Items → Stockpile an Station (CM)
              ↓
            Sale → Sale Items (FIFO aus storage_items)
              ↓
           Payout → Payout Items

    • Jede Materialerfassung erzeugt einen Batch + Lager-Eintrag auf dem Schiff
    • Standort-Seite (material_stockpiles): WO liegt wie viel — Station/Stadt/Schiff
    • Raffinerie verarbeitet Batches; Rohmaterial vom Schiff, CM als Stockpile
    • Verkäufe entnehmen aus storage_items (FIFO); Dashboard ohne Doppelzählung
    • Auszahlungen verknüpfen Verkäufe mit Sessions über Lager-Trace
    • Verkaufbare Materialien: RMC, CM (raffiniert)
    • Rohmaterial (CM Rubble/Scraps/Salvage): nur Raffinerie-Input

2.3 Host-Client (ab v0.10.0, CREW Edition ab v0.15.0)
    Host (SQLite) ←TCP/JSON→ Client(s) (RemoteDatabase / RPC)
    • Ein Rechner = Host (Quelle der Wahrheit)
    • Clients: Sitzung wählen, Material/Raffinerie/Verkauf je nach Rechten
    • Standalone = SOLO Edition (Standard-Download)

2.4 Produkt-Editionen (ab v0.15.0 Beta)
    SOLO  — SC Salvage Tracker - SOLO Version (kostenlos, Standard)
    CREW  — Vernetzung Host/Client (Build oder später Supporter-Key)
    ORGA  — Organisations-Modul (geplant)
    Siehe Changelogs/EDITIONS.txt
